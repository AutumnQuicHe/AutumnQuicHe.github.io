QUIC全集

QUIC是新一代传输协议。

QUIC工作组（QUIC Work Group, quicwg）是QUIC协议撰写小组，其站点详细列出了围绕该协议已发版或正在撰写阶段的全部IETF文档及草稿。

本站翻译其中已经定稿的文档并予以展示。

QUIC四层

已经定稿的QUIC四层文档共8篇，原文及中译如下：

RFC8999：《QUIC版本通用属性（Version-Independent Properties of QUIC）》

中译：《RFC8999中译：QUIC版本通用属性》。

RFC8999定下了QUIC协议版本升级及协商所需要遵守的规范，后世所有QUIC版本均须遵守，故译“QUIC版本通用属性”。

RFC9000：《QUIC，一种基于UDP的安全多路复用传输协议（QUIC: A UDP-Based Multiplexed and Secure Transport）》

中译：《RFC9000中译：QUIC传输协议》

RFC9000详细描绘了QUIC四层可靠传输协议，一般简称为“QUIC传输协议”。同时，由于该文是自QUIC问世以来的发版的第一份RFC正式文档，因此其定义的QUIC协议也称被为“QUIC第1版”。

RFC9001：《使用TLS保护QUIC（Using TLS to Secure QUIC）》

中译：《RFC9001中译：QUIC TLS》

RFC9001描述了如何使用TLS加密QUIC。

RFC9002：《QUIC丢包检测和拥塞控制（QUIC Loss Detection and Congestion Control）》

中译：《RFC9002中译：QUIC恢复》

RFC9002描述了QUIC的丢包检测与恢复算法有关内容，其中提出了PTO机制用以取代RTO和TLP机制管理数据包超时重传。

RFC9221：《QUIC不可靠数据报文传输扩展（QUIC Loss Detection and Congestion Control）》

中译：《RFC9221中译：QUIC不可靠数据传输》

RFC9221描述了如何使用QUIC进行不可靠数据传输。

RFC9368：《QUIC兼容版本协商（Compatible Version Negotiation for QUIC）》

中译：《RFC9368中译：QUIC兼容版本协商》

具体定义了QUIC的版本协商机制，包括双端如何协商出一个兼容版本。

RFC9369：《QUIC第2版（QUIC Version 2）》

中译：《RFC9369中译：QUIC第2版》

发布了QUIC第2版。

RFC9443：《QUIC的多路复用更新（Multiplexing Scheme Updates for QUIC）》

中译：《RFC9443中译：QUIC多路复用》

描述QUIC针对多路复用进行的更新。
